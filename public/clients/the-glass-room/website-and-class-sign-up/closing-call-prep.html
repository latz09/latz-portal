Response Scenarios

Pricing
"$10,500 is more than we expected" — First Response
⤴
  • Hold the line — don't offer adjustments yet
  • Walk through what the signup system actually replaces
  • Frame the build complexity honestly
  • Let them tell you what they want to cut
"What if we skip the private event pages?" — They Pick a Lever
⤴
  • This is good — they're working with the proposal
  • Confirm the savings (~$1,200) and what changes
  • Explain the SEO trade-off honestly
  • Single page + dropdown still captures leads
"Can we cut more? What about the studio sub too?" — Stacking Cuts
⤴
  • Both are deferrable — not gone forever
  • Brings it to ~$8,500
  • PDF menu swap available as a third cut (-$400)
  • All three = $8,100 — that's the visible bottom
"We were hoping closer to $6,000–$7,000" — Below Adjustments
⤴
  • Acknowledge the budget honestly
  • $7,500 is the absolute floor — all adjustments + concession
  • Below that, scope doesn't support the signup system quality
  • The signup system is 40–50% of this project's value
"We can't go above $7,500" — At the Floor
⤴
  • This is still a good project — take it
  • All three proposal adjustments applied
  • Plus one additional concession to bridge the gap
  • Be clear: Phase 2 items are priced separately later
Goes quiet / needs to "think about it" — Drop the Seeds
⤴
  • Use the timeline angle — mid-June is coming fast
  • Monthly fee math still works here
  • The signup system is the revenue engine
  • Don't pressure — but plant the urgency
Scope & Technical
"Why is the signup system so expensive?"
⤴
  • This isn't a contact form — it's a booking engine
  • Stripe, waivers, capacity, waitlist, emails
  • Admin dashboard with CSV export
  • Alternative: Acuity/Calendly at $30+/mo forever
Derek questions scope or suggests DIY parts
⤴
  • Respect his background — don't be defensive
  • Marketing/design ≠ custom web development
  • The CMS is built for Sam's workflow specifically
  • Welcome his input on design direction
"Can you guarantee it's done by mid-June?"
⤴
  • 2–3 months from deposit is realistic
  • Wireframe done + branding handled = head start
  • Signup system needs thorough testing — can't rush it
  • Rush fee available: +25% if needed
"Why not Wix / Squarespace?"
⤴
  • Class signup system won't work on those platforms
  • Custom waiver + capacity + waitlist + Stripe = custom code
  • Monthly fees add up — $300+/yr on Squarespace vs $240/yr Vercel
  • You own this. No platform lock-in.
"What about the email marketing piece?"
⤴
  • Separate from transactional class emails (those are built in)
  • Mailchimp, Kit, Beehiiv — they pick the platform
  • ~$10–30/mo depending on list size
  • Integration is included in the build
"We don't have photos of the space yet"
⤴
  • Sam's a photographer — staged shots from her studio work
  • Stock as placeholders, swap real photos at no cost
  • Real class photos post-launch make the site stronger
  • Build-out photos could work for a "coming soon" feel
Process & Payment
"Can we do a different payment split?"
⤴
  • 50/50 is standard and non-negotiable
  • Deposit locks the timeline slot
  • Second half at launch — they see the finished product first
"What happens next?"
⤴
  • Green light → simple agreement + invoice
  • 50% down → kickoff within 24 hours
  • First draft core pages in a few weeks
  • Live dev link — they watch it come together
"What about Phase 2 / online sales?"
⤴
  • Online sales deferred to Phase 2 — smart move
  • Gallery with Etsy linking covers launch
  • Phase 2 priced separately once they know demand
  • Any deferred items can be added without rebuilding
