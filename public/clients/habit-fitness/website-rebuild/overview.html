Feature · v1

Tutorial Video System

  • QR on every machine → branded page at /m/[slug] (not raw YouTube)
  • Public library — sticky search + faceted filters (muscle group AND/OR equipment)
  • One CMS "Machine/Video" dataset powers both QR pages + library
  • Priced as an optional bundled line item
Two independent facets + searchable aliases is what makes the library findable.

Muscle group + equipment type as two separate filters (not one tree) plus alias search is what lets the library scale from 30 to 300 machines without the filters ever getting messy.

🧭
Central library decision: TWO independent facets + free-text search — NOT a category tree A single hierarchy forces a machine like the lat pulldown to live under either "Back" or "Cable" when it's both. Faceted filtering lets a member filter by muscle group AND/OR equipment type and find it either way. This is the core design decision for the library — surface it in the wireframe.

Video Delivery & QR

Embeds Unlisted YouTube via youtube-nocookie.com, rel=0, playsinline=1. Facade/lite-embed (branded thumbnail, load iframe on tap) for speed.
YT Chrome modestbranding deprecated — can reduce but not fully remove YouTube chrome on iframe. (Premium host upgrade is a future upsell.)
QR Short path /m/<slug> for fast scan. Error-correction level Q for durability. Laminated/vinyl stickers.

CMS Data Model — "Machine/Video" Content Type

title String — machine/exercise name
slug String — drives the QR URL. Flat and human (e.g. lat-pulldown). NEVER category-prefixed, so recategorizing never breaks a printed QR.
muscleGroup Single-select, LOCKED enum — Axis 1 / primary filter
equipmentType Single-select, LOCKED enum — Axis 2 / secondary filter
secondaryTags Multi-select — other muscles worked. Feeds search/relevance, NOT shown as a filter.
aliases String list — alternate names for search only (e.g. "pulldown", "lat machine")
videoId String — Unlisted YouTube ID
description Rich text, optional, blank by default
Search Matches Across title + aliases + secondaryTags

Locked Enum — muscleGroup (Axis 1)

Chest · Back · Shoulders · Arms · Legs · Glutes · Core · Cardio · Full Body / Functional

Glutes is intentionally split out from Legs — heavily searched, especially for the growing women's demographic.

Locked Enum — equipmentType (Axis 2)

Selectorized machine · Plate-loaded · Cable · Free weight · Smith machine · Cardio · Functional / Bodyweight

⚠️
BUILD RULE: muscleGroup + equipmentType must be locked dropdowns — never free-typed by staff If staff can type them, the filters fragment ("Back" vs "back" vs "Lats") and the library breaks. Locked enums also keep staff entry idiotproof, per Andy's ask.

Edge Cases

Duplicate Machines e.g. three identical leg presses → ONE record. All identical QR stickers point to the same slug. Do not create duplicates.
Multi-Exercise Stations Cable crossover, functional trainer → v1: one QR / one page per physical station, key movements timestamped within the single video. Per-exercise entries = v2 expansion.

Requirements

Mobile-First Every QR scan is on a phone at a machine — mobile is the primary surface
Performance Fast load — facade embeds, optimized media
Visual Quality Premier/elite, dark direction (black / white-grey / dark-military-tan accent)
UX Simple, idiotproof — minimal friction to "come in"
Constraint No AI anywhere — including spam protection
