Rod Builder — Implementation Details

The rod builder is the core differentiator of this site — get it right

Valid options only (no impossible combos). Real-time price updates. Server-side price calculation (secure). All selections land in Shopify dashboard as structured data.

Rod Builder Requirements

Frontend Custom React component — step-by-step selector or single-page configurator (TBD with Chris)
Validation Only show valid combinations — prevents impossible builds that caused issues in v1
Pricing Real-time price display on frontend — but final price calculated server-side before creating draft order
Selections Needed Lengths, blanks, colors, finishes, guides, hardware — full list from Chris (pending)
Output Structured draft order via Admin API — every selection is a line item or property, not a note
Pseudo-flow — Rod Builder to Checkout 
// 1. Customer completes rod configuration on frontend
const rodConfig = {
  blank: "St. Croix SCIII 7'0\" Medium",
  guides: "Fuji Alconite",
  handle: "Full cork grip",
  finish: "Metallic blue thread wrap",
  // ... all selections
}

// 2. Send to Next.js API route (server-side)
const response = await fetch('/api/create-rod-order', {
  method: 'POST',
  body: JSON.stringify(rodConfig)
})

// 3. Server validates selections + calculates price
// 4. Server calls Shopify Admin API → creates draft order
// 5. Shopify returns checkout URL for that draft order
// 6. Redirect customer to Shopify checkout URL
// 7. Order completes → lands in Chris's dashboard, fully structured
🔐
Security: Price must be calculated server-side Never trust frontend-calculated prices. The Next.js API route validates the config and computes the final price before sending to Shopify. This was a weakness in Option 1 (line item properties approach) — don't repeat it.
📦

Why Draft Orders (Option 2)

🧠
This was the key architectural decision — document the reasoning Two options were considered for getting custom rod builds into Shopify checkout. Option 2 (draft orders) was chosen. Here's why.
Criteria Option 1 — Line Item Properties Option 2 — Draft Orders via Admin API
How it works One "Build Your Rod" product in Shopify. Selections passed as notes on the line item. Selections sent as structured data via Admin API. Creates a real draft order.
Price calculation Calculated on frontend — minor security concern Calculated server-side — secure
Order structure Selections are unstructured notes — harder to parse Clean, structured data — every selection is a field
Build complexity Simpler to implement Slightly more complex — requires Admin API
Future scaling Hard to build reporting or automations on top of note data Ready for reporting, automation, integrations
Verdict Rejected — too fragile for a business that processes orders daily ✅ Chosen — right foundation long-term
🎯
Why it mattered for Chris specifically His previous setup was Square for payments + a third-party shipping tool — fragmented, manual, multiple platforms per order. Shopify consolidates everything. Draft orders ensure custom rod orders are just as clean and structured as standard product orders. He processes an order, sees exactly what was built, prints a label, ships — all one place.

Build Notes & Reminders

API Keys — Two Different APIs

Storefront API (GraphQL) is used for reading product data and standard cart/checkout. Admin API is used only for the rod builder draft orders — it has write access. Keep these separated mentally. Storefront API keys are public-safe (they go to the browser). Admin API keys are secret (server-side only, never exposed to frontend).

Draft Order Checkout URL

When you create a draft order via Admin API, the response includes an invoice_url — that's the hosted checkout URL for that specific order. Redirect the customer there. Don't try to build your own checkout. Shopify handles everything from that point: payment, tax, shipping, confirmation.

Product Images — Lesson from V1

V1 had white-background product images that looked bad on the custom layout. The photography shoots for the rebuild should deliver transparent-background images. Confirm this with Chris / photographer before the shoots happen. This is a design-critical detail.

Sanity Schema Design

Design Sanity schemas so Chris can add new product categories, update marketing blocks, swap photography — all without calling Jordan. The whole point of v2 is that Chris runs the content. If a schema requires developer intervention to add a new product type, you failed.

Shopify Partner Dev Store — No Time Limit

Free development stores through the Partner program have no time limit and no charge. Don't rush the transfer. Only transfer when the site is fully tested and ready for real transactions. Chris doesn't start paying $39/month until the store is transferred and he picks a plan.

V1 Pricing Context

V1 was $4,025 — intentionally underpriced to break into the WI market. V2 is $4,500–$5,500 for what's honestly a more complex build (Shopify integration, rod builder, dual checkout flows). The price is fair. Don't undersell it.
