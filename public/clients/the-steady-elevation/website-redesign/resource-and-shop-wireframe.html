Payment Backend — Major Decision
Decision — Payment Backend
Option A — Lemon Squeezy (Recommended for v1)
External hosted checkout. Lemon Squeezy handles payment, file delivery, and sales tax compliance (they're the merchant of record). Simpler build, faster launch. Fee: 5% + 50¢ per sale.
Option B — Custom Stripe
Files live in the CMS, automated email delivery via webhook, single dashboard alongside everything else. More build time. Fee: 2.9% + 30¢ per sale — you handle sales tax compliance yourself.
The visual layout of resource cards is identical either way — only the backend differs. Lemon Squeezy gets you live faster; Stripe is cheaper per-sale long-term once volume justifies the build.

Notify Me
Stephanie toggles "Collect Interest" on for any event in the CMS. The form appears below the event description when the toggle is on. Submitted emails are stored on the event document — she can export the full interest list as a CSV anytime from the CMS. No third-party email platform needed; the list lives with the event so it's tied to the right audience.
Decision — Interest Form Placement
Option A — Inline on the card (shown above)
Form is always visible on the event card. Lower friction, more visible — visitors see the option without clicking. Card is a bit taller.
Option B — Expand/modal on button click
Card stays compact with a "Notify Me" button. Clicking expands the form (or opens a small modal). Cleaner card, one extra click.
Either works. Option A is recommended for the lower friction. Designer's call after seeing it in context.
